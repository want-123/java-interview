tools权限使用langgraph4j优化：
1. 权限校验时机：大模型决策后
每个子 Agent 先通过大模型（langchain4j）决策要调用的具体 Tool ID（如distributed_lock_tool）；
仅在决策出具体 Tool 后，才调用checkToolPermission(userId, toolId)校验该 Tool 的权限；
完全符合你 “大模型决定调用哪一个具体的 tools 时才校验权限” 的核心需求。
2. 多 Agent 并行的权限独立性
每个子 Agent 独立决策 Tool、独立校验权限，互不干扰；
比如 JVM Agent 决策调用jvm_gc_tool（所有用户有权限），分布式 Agent 决策调用distributed_lock_tool（仅 VIP 有权限），权限结果各自独立。
3. 状态可追溯与恢复
每个子 Agent 的 “决策 Tool ID”“权限结果”“执行状态” 都持久化到 Checkpointer；
即使任务中断，恢复后可从 “大模型决策 Tool” 或 “权限校验” 节点继续执行，无需重新决策。
4. 复用现有逻辑
完全复用你现有 langchain4j 的：大模型决策 Tool 逻辑、Tool 执行逻辑、权限控制逻辑；
langgraph4j 仅负责 “流程编排 + 状态管理”，无侵入式修改。

面试：
1，基于 LangGraph4j 构建问题分解流程，拆解复杂多跳问题为可执行子任务，提高问题解决成功率。
使用langgraph将复杂问题按照：问题分解节点（保存子问题列表到全局状态），子问题执行节点，结果聚合整理节点
2.利用Langgraph4j全局状态机+ABAC实现多Agent异步调用环境下的Tools调用权限控制，保障系统数据安全与操作合规。
问题分解节点将复杂问题分解为多个小问题同时给出子问题所需的tools列表，执行问题节点中Agent通过全局状态获取tools列表进行权限判断
ABAC权限：主体(Agent)、客体(tools)、行为(主体对客体的的操作行为)、环境(时间、ip、系统等)
